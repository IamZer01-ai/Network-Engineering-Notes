# Networking Fundamentals

## Introduction

Networking is the practice of connecting computers, servers, mobile devices, and other systems to enable communication and resource sharing. Modern organizations rely on networks for data exchange, internet access, cloud services, and collaboration.

A network can range from a small home setup with a few devices to a global infrastructure connecting millions of systems across the internet.

## Why Networking is Important

* Enables communication between devices
* Allows sharing of resources such as printers and files
* Provides internet connectivity
* Supports cloud computing
* Facilitates remote access and collaboration
* Forms the foundation of cybersecurity operations

## Basic Components of a Network

### End Devices

Devices that send or receive data.

Examples:

* Computers
* Laptops
* Smartphones
* Servers
* Printers

### Network Devices

#### Router

Connects multiple networks and forwards packets between them.

Functions:

* Routes traffic
* Connects LANs to the Internet
* Performs NAT

#### Switch

Connects devices within a Local Area Network (LAN).

Functions:

* Learns MAC addresses
* Forwards frames
* Supports VLANs

#### Access Point

Provides wireless connectivity to devices.

Functions:

* Extends network access
* Enables Wi-Fi communication

### Transmission Media

#### Wired Media

* Ethernet Cable
* Fiber Optic Cable
* Coaxial Cable

#### Wireless Media

* Wi-Fi
* Bluetooth
* Cellular Networks

## Types of Networks

### LAN (Local Area Network)

A network covering a small geographical area such as a home, office, or campus.

Characteristics:

* High speed
* Low latency
* Privately managed

### WAN (Wide Area Network)

A network that spans large geographical areas.

Examples:

* Internet
* Enterprise WAN

### MAN (Metropolitan Area Network)

A network covering a city or metropolitan region.

### PAN (Personal Area Network)

A short-range network connecting personal devices.

Examples:

* Bluetooth devices
* Smart watches

## Network Topologies

### Bus Topology

All devices share a common communication cable.

### Star Topology

All devices connect to a central switch.

Advantages:

* Easy troubleshooting
* Better performance

### Ring Topology

Devices form a circular connection.

### Mesh Topology

Every device connects to multiple devices.

Advantages:

* High redundancy
* Fault tolerance

## Data Communication Process

1. Sender creates data.
2. Data is converted into packets.
3. Packets travel through switches and routers.
4. Destination receives packets.
5. Packets are reassembled into original data.

## Key Networking Terms

### Bandwidth

Maximum amount of data that can be transmitted per second.

### Latency

Time required for data to travel from source to destination.

### Throughput

Actual data transfer rate achieved.

### Packet

A unit of data transmitted over a network.

### Protocol

A set of rules governing communication between devices.

## Common Networking Protocols

* TCP
* UDP
* IP
* ICMP
* ARP
* HTTP
* HTTPS
* DNS
* DHCP
* SSH

* Centralized management
